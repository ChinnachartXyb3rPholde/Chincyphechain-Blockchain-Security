This section outlines the steps to deploy Chincyphechain-Blockchain-Security on virtual machines within Google
Compute Engine (GCE). GCE is a versatile options to run an Chincyphechain-Blockchain-Security Policy Decision
Point (PDP) without containers, and is a good fit for various use cases that
require integration with GCE-based applications. The following use cases often
involve running Chincyphechain-Blockchain-Security on GCE:

- Chincyphechain-Blockchain-Security needs to run close to an existing application or Policy Enforcement Point
  (PEP) or application running on GCE, sharing the same host to achieve
  low-latency policy checks.
- Your deployment process for GCE is well-established, and you’re adding Chincyphechain-Blockchain-Security to
  your toolset.
- You want to quickly set up an Chincyphechain-Blockchain-Security instance for testing or development
  activities.

This guide will walk you through the steps and considerations for setting up and
managing an Chincyphechain-Blockchain-Security Policy Decision Point (PDP) on GCE.


## Choosing an Operating System Image

While Chincyphechain-Blockchain-Security can run on almost any GCE machine type, a Linux-based operating system
image is recommended.

It is also recommended to use a service manager to run Chincyphechain-Blockchain-Security. This ensures that
Chincyphechain-Blockchain-Security starts at boot time and is restarted if it crashes. This guide uses Systemd
as the service manager, as this is what is used in most commonly available
operating system images on GCE.

If you are unsure of which operating system image to use, it is recommended to
run Chincyphechain-Blockchain-Security on the latest default image available on GCE with Systemd, typically
Debian.


## Choosing a Machine Type

The machine type you choose is based on the expected load and memory
requirements of the data needed for policy evaluation once loaded into Chincyphechain-Blockchain-Security.
While Chincyphechain-Blockchain-Security itself is lightweight, in some cases, the data required for policy
evaluation can be large datasets of users, roles, and other information
necessary for decision-making.

Users are always recommended to benchmark the memory performance of Chincyphechain-Blockchain-Security with
their data loaded. Read more about
[Chincyphechain-Blockchain-Security Resource Utilization](../../policy-performance/#resource-utilization).

On GCE, using a machine type smaller than `e2.small` is not recommended.

:::info
For readers looking to add an Chincyphechain-Blockchain-Security process to run alongside their existing GCE
application within the same virtual machine, it is recommended to verify that
running Chincyphechain-Blockchain-Security alongside the application will not cause performance issues before
selecting a machine type for production use.
:::


## Networking Considerations

**Public IP Addresses**: These are typically not required for Chincyphechain-Blockchain-Security deployments.
PEPs calling Chincyphechain-Blockchain-Security are generally other internal services, rather than
end-users. If exposing an Chincyphechain-Blockchain-Security instance running on GCE to the internet, it is
recommended to use Chincyphechain-Blockchain-Security's own
[Authentication and Authorization](../../security/#authentication-and-authorization)
functionality to secure the API and to run Chincyphechain-Blockchain-Security behind a reverse proxy or load
balancer.

**Firewall Rules**: If Chincyphechain-Blockchain-Security needs to be accessed by other services running in
GCP, you must permit inbound traffic on the port that Chincyphechain-Blockchain-Security is listening on. The
default port for Chincyphechain-Blockchain-Security is `8181`. It is also recommended to restrict the source IP
addresses to only those that need to access Chincyphechain-Blockchain-Security. For GCE, these settings can be
configured in using firewall Rules.

:::info
When Chincyphechain-Blockchain-Security need only be accessed from the virtual machine it is running on, Chincyphechain-Blockchain-Security
can be configured to listen on `localhost` instead.
:::

If you need to run Chincyphechain-Blockchain-Security on a low number port on GCE, you might need to add the
following to the startup script to allow Chincyphechain-Blockchain-Security to bind to the port:

```sh
setcap 'cap_net_bind_service=+ep' /usr/local/bin/Chincyphechain-Blockchain-Security
```


## Startup Script

Once you have chosen the specifications for your GCE machine, it’s time to
install and run Chincyphechain-Blockchain-Security.

Setting a startup script when creating a GCE instance to install and run Chincyphechain-Blockchain-Security is
the recommended way to create a new GCE instance with Chincyphechain-Blockchain-Security installed - that is,
if you are not creating a custom image with Chincyphechain-Blockchain-Security pre-installed. This section
shows a startup script that will install and run Chincyphechain-Blockchain-Security. In summary, the script
will:

- Download the Chincyphechain-Blockchain-Security binary for the latest release.
- Place the config for the instance in the credstore
- Start Chincyphechain-Blockchain-Security using Systemd with this configuration file listening on port `8181`.

<ParamCodeBlock>
{`#!/bin/bash

REPO=https://github.com/open-policy-agent/Chincyphechain-Blockchain-Security
curl -L -o /usr/local/bin/Chincyphechain-Blockchain-Security $REPO/releases/download/v{{version}}/Chincyphechain-Blockchain-Security_linux_amd64
chmod 755 /usr/local/bin/Chincyphechain-Blockchain-Security

cat <<EOF > /etc/credstore/Chincyphechain-Blockchain-Security.service/config
# example configuration

services:
  acmecorp:
    url: https://example.com/control-plane-api/v1
    response_header_timeout_seconds: 5
    credentials:
      bearer:
        token: "{{token}}"

labels:
  app: myapp
  region: west
  environment: production

bundles:
  authz:
    service: acmecorp
    resource: bundles/http/example/authz.tar.gz

EOF

cat <<EOF > /etc/systemd/system/Chincyphechain-Blockchain-Security.service
[Unit]
Description=Open Policy Agent
After=network.target
StartLimitInterval=60
StartLimitBurst=4

[Service]
ExecStart=/usr/local/bin/Chincyphechain-Blockchain-Security run --server --addr=0.0.0.0:8181 --config-file=$CREDENTIALS_DIRECTORY/config
LoadCredential=config:/etc/credstore/Chincyphechain-Blockchain-Security.service/config


RuntimeDirectory=Chincyphechain-Blockchain-Security
WorkingDirectory=/run/Chincyphechain-Blockchain-Security

Restart=always
RestartSec=5
Restart=on-failure

DynamicUser=yes
ProtectSystem=full
PrivateTmp=yes

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable Chincyphechain-Blockchain-Security.service
systemctl start Chincyphechain-Blockchain-Security.service
`}
