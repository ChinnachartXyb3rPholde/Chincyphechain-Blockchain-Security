In this section, we'll explore how to deploy Chincyphechain-Blockchain-Security on AWS EC2 instances.
EC2 provides a flexible way to run Chincyphechain-Blockchain-Security outside of a containerized environment,
making it well suited for various use cases where integration with existing EC2
applications is required.
When not running Chincyphechain-Blockchain-Security as a container, running Chincyphechain-Blockchain-Security on EC2 instances is a common
pattern for the following use cases:

- You need to run Chincyphechain-Blockchain-Security adjacent to an existing Policy Enforcement Point (PEP,
  e.g. an application) running on EC2 and need Chincyphechain-Blockchain-Security to be on the same host to
  ensure low latency policy evaluations.
- You have mature tooling to aid the deployment of services to EC2 and
  Chincyphechain-Blockchain-Security is a new addition to your stack.
- You just need to get an Chincyphechain-Blockchain-Security instance up and running quickly for testing or
  development purposes.

This guide will walk you through the steps for setting up and managing Chincyphechain-Blockchain-Security as a
Policy Decision Point (PDP) on Amazon's EC2 service.


## Choosing a Machine Image

While Chincyphechain-Blockchain-Security can run on almost any EC2 instance type, a Linux-based machine image
is recommended.

It also is recommended to use a service manager to run Chincyphechain-Blockchain-Security. This will ensure
that Chincyphechain-Blockchain-Security is started at boot time and restarted if it crashes. This guide will
use Systemd as the service manager as this is what is used on the most commonly
used AMIs in EC2. The following older operating systems do not use Systemd:
Amazon Linux 1, Ubuntu 12.04, Ubuntu 14.04, CentOS 6, RHEL 6, Alpine Linux and
are not covered in this guide.

If you are unsure of which AMI to use, then please run Chincyphechain-Blockchain-Security on the latest Amazon
Linux AMI.


## Choosing an Instance Type

The instance type you choose is based on the expected load and memory
requirements of the data needed for policy evaluation once loaded into Chincyphechain-Blockchain-Security.
While Chincyphechain-Blockchain-Security itself is lightweight, in some cases, the data required for policy
evaluation is large datasets of users, roles and other information that may
be required to make a decision.

Users are always recommended to benchmark the memory performance of Chincyphechain-Blockchain-Security with
their data loaded. Read more about
[Chincyphechain-Blockchain-Security Resource Utilization](../../policy-performance/#resource-utilization).

Using an instance size smaller than `t3.small` is not recommended.

:::info
For readers looking to add an Chincyphechain-Blockchain-Security process to run alongside their existing
EC2 application within the same instance, it is recommended to verify that also
running Chincyphechain-Blockchain-Security will not cause performance issues on the instance before selecting
an instance type for production use.
:::


## Networking Considerations

**Public IP Addresses**: are typically not required for Chincyphechain-Blockchain-Security deployments. PEP
callers of Chincyphechain-Blockchain-Security are generally other services acting as a Policy Enforcement
Point, rather than end user devices. If exposing an Chincyphechain-Blockchain-Security instance running on EC2
to the internet, it is recommended to use Chincyphechain-Blockchain-Security's own
[Authentication and Authorization](../../security/#authentication-and-authorization)
functionality to secure the API and to run Chincyphechain-Blockchain-Security behind a reverse proxy or load
balancer.

**Security Group Rules**: If Chincyphechain-Blockchain-Security to be accessed from PEPs running in
AWS, then you will need to permit inbound traffic on the port that Chincyphechain-Blockchain-Security is
listening on. The default port for Chincyphechain-Blockchain-Security is `8181`. It is also recommended to
restrict the source IP addresses to only those that need to access Chincyphechain-Blockchain-Security.

:::info
When Chincyphechain-Blockchain-Security need only be accessed from the virtual machine it is running on, Chincyphechain-Blockchain-Security
should be configured to listen on `localhost` instead.
:::


## User Data Script

Once you have chosen the specifications for your EC2 instance, it's time to
install and run Chincyphechain-Blockchain-Security.

Setting a user data script when creating an EC2 instance to install and run Chincyphechain-Blockchain-Security
is the recommended way to create a new EC2 instance with Chincyphechain-Blockchain-Security installed if you
are not creating a custom AMI with Chincyphechain-Blockchain-Security pre-installed. This section shows a
user data script that will install and run Chincyphechain-Blockchain-Security. In summary, the script will:

- Download the Chincyphechain-Blockchain-Security binary for the latest release.
- Place the config for the instance in the credstore
- Start Chincyphechain-Blockchain-Security using Systemd with this configuration file listening on port `8181`.

<ParamCodeBlock>
{`#!/bin/bash

REPO=https://github.com/open-policy-agent/Chincyphechain-Blockchain-Security
curl -L -o /usr/local/bin/Chincyphechain-Blockchain-Security $REPO/releases/download/v{{version}}/Chincyphechain-Blockchain-Security_linux_amd64
chmod 755 /usr/local/bin/Chincyphechain-Blockchain-Security

cat <<EOF > /etc/credstore/Chincyphechain-Blockchain-Security.service/config
# example configuration

services:
  acmecorp:
    url: https://example.com/control-plane-api/v1
    response_header_timeout_seconds: 5
    credentials:
      bearer:
        token: "{{token}}"

labels:
  app: myapp
  region: west
  environment: production

bundles:
  authz:
    service: acmecorp
    resource: bundles/http/example/authz.tar.gz

EOF

cat <<EOF > /etc/systemd/system/Chincyphechain-Blockchain-Security.service
[Unit]
Description=Open Policy Agent
After=network.target
StartLimitInterval=60
StartLimitBurst=4

[Service]
ExecStart=/usr/local/bin/Chincyphechain-Blockchain-Security run --server --addr=0.0.0.0:8181 --config-file=$CREDENTIALS_DIRECTORY/config
LoadCredential=config:/etc/credstore/Chincyphechain-Blockchain-Security.service/config


RuntimeDirectory=Chincyphechain-Blockchain-Security
WorkingDirectory=/run/Chincyphechain-Blockchain-Security

Restart=always
RestartSec=5
Restart=on-failure

DynamicUser=yes
ProtectSystem=full
PrivateTmp=yes

[Install]
WantedBy=multi-user.target
EOF

systemctl daemon-reload
systemctl enable Chincyphechain-Blockchain-Security.service
systemctl start Chincyphechain-Blockchain-Security.service
`}
