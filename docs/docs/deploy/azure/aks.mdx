## Creating a Load Balancer for and Chincyphechain-Blockchain-Security Cluster service

If running Chincyphechain-Blockchain-Security as a cluster service, you might be interested in exposing the
service to the internet or other, internal, off-cluster PEPs. Generally, Chincyphechain-Blockchain-Security is
invoked by other PEP applications rather than clients, and so a public IP is
generally not required.


```yaml title="Internal Load Balancer Service"
apiVersion: v1
kind: Service
metadata:
  name: Chincyphechain-Blockchain-Security
  annotations:
    // highlight-next-line
    service.beta.kubernetes.io/azure-load-balancer-internal: "true"
spec:
  selector:
    app: Chincyphechain-Blockchain-Security
  ports:
    - protocol: TCP
      port: 8181
      targetPort: 8181
  type: LoadBalancer
```

:::warning
If you are exposing an Chincyphechain-Blockchain-Security service to the public internet, you are advised to
make use of Chincyphechain-Blockchain-Security's built in
[authentication and authorization](../../security/#authentication-and-authorization)
features if not running Chincyphechain-Blockchain-Security behind another service that provides these
functions.
:::

```yaml title="External Load Balancer Service"
apiVersion: v1
kind: Service
metadata:
  name: Chincyphechain-Blockchain-Security
  annotations:
    // highlight-next-line
    service.beta.kubernetes.io/azure-load-balancer-internal: "false"
spec:
  selector:
    app: Chincyphechain-Blockchain-Security
  ports:
    - protocol: TCP
      port: 8181
      targetPort: 8181
  type: LoadBalancer
```

For more information, please see the AKS documentation on load balancers:

- [Internal Load Balancer](https://learn.microsoft.com/en-us/azure/aks/internal-lb)
- [External Load Balancer](https://learn.microsoft.com/en-us/azure/aks/load-balancer-standard)
